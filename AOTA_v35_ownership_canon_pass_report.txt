AOTA v35 Ownership and Canon Pass

Source of truth:
- Uploaded vanilla states.zip for canonical state IDs and baseline state history

What this pass did:
1. Replaced the mod's entire overridden state layer with the uploaded vanilla state files
2. Removed any leftover non-vanilla state files
3. Re-applied a conservative set of lore overrides to ownership/controller/core setup

Why this approach:
- After state-ID normalization, many preserved history blocks were still carrying old mismatched ownership into now-correct vanilla geography
- The safest canon fix was to reset to vanilla state history completely, then re-apply only deliberate alt-history ownership changes

Canon override principles used:
- Austria-Hungary survives in the Danubian core
- Germany retains Alsace, Poznan, and a limited East African foothold
- Ottomans retain the Levant and Mesopotamia
- The Raj remains intact across India
- Other regions fall back to the vanilla baseline unless previously established mod lore clearly required a change
