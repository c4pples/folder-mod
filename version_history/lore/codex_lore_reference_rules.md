# Codex Lore Reference Rules (AOTA)

## Purpose
This file defines mandatory usage rules for future Codex passes that touch narrative/game-state content.

## Primary reference
Before making major content edits, read:
- `version_history/lore/canon_game_start_lore_bible.md`

Treat that file as the default canon baseline for game start unless a later intentional canon update explicitly supersedes it.

## Required workflow for lore-sensitive changes
1. Identify which section(s) of the lore bible apply (country, region, ideology, actors, tone).
2. Confirm your change is an extension of existing canon, not contradiction.
3. Preserve ambiguity where the bible or source content marks ambiguity as intentional.
4. Keep institutions, actor coalitions, and rhetorical style consistent with established setting logic.
5. In your final report/commit note, state whether your work:
   - preserves canon,
   - extends canon conservatively, or
   - deliberately revises canon (must be explicit).

## Must-preserve elements
- Unfinished-peace framing and legitimacy crisis tone.
- Major pressure systems (Rhine, Danube, Russian reunification, Ottoman provincial strain, U.S. constitutional rupture risk, imperial overstretch).
- Character agency logic (leaders/advisors are historical actors with constituencies, not generic modifiers).
- Regional rhetorical distinctions.

## Prose guidance summary
- Serious, political, institution-aware prose.
- Avoid meme/comic style, flat boilerplate, and ahistorical caricature.
- Make conflicts feel driven by people, networks, and governing structures.

## If adding new lore
When you introduce material not already explicit in the bible, document:
- what evidence implied it,
- why it is conservative and setting-consistent,
- what remains uncertain.

## Enforcement note
If you modify major focuses/events/spirits/characters without consulting the lore bible, that pass should be considered incomplete and requires revision.
